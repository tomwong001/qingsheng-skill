# 我把 Claude Code 变成了中文恋爱 AI：一个叫「情圣」的开源 skill 是怎么做出来的

**主打关键词**：Claude Code skill、中文恋爱 AI、情圣、情感教练、微信聊天助手、恋爱话术、AI 约会助手

## 一、为什么跟女生聊天这么难

先说一个我自己的真实经历。

去年冬天我在探探上匹配到一个看起来很聊得来的女生，聊了三天，氛围一直不错。第四天她发来一句："今天下雨了，有点想喝奶茶。"

我盯着这句话看了十分钟。

这是在暗示我请她喝奶茶？还是单纯分享心情？如果是暗示，我直接问"要不要一起"会不会太冒进？如果是分享，我回一句"注意保暖"会不会显得情商低？我最后回了一个"我这边也下了，记得带伞"——非常安全，非常无聊，非常典型的**聊着聊着就消失**。

后来复盘，我意识到一个残酷的事实：**在中文语境下跟女生聊天，本质上是在做高维度的社交信号解码**。一句话背后往往有三层含义：字面意思、情绪状态、关系推进意图。而绝大多数男生——包括当时的我——只读到了第一层。

这不是智商问题，是训练数据不够。女生从小被社交环境"喂"了大量隐性沟通样本，男生普遍没有。

## 二、现有方案为什么都不行

我试过三种方案，都失败了。

**方案一：付费恋爱课程**。几千到几万不等，教的东西基本停留在 2015 年 PUA 那套——"冷读热读"、"推拉框架"、"负面反馈"。问题是这套话术在 2020 年之后已经被女生识别成"油腻红灯"，现在用等于自爆。而且课程是录播的，没法针对你当前这段具体聊天给建议。

**方案二：裸用 ChatGPT 或 Claude**。看起来免费又强大，实际效果很差。你把聊天记录贴进去，它给你的回答会是这样的：

> "建议以真诚和尊重为基础进行沟通。可以表达关心，询问对方的感受，保持开放和积极的态度。记得每个人都是独特的个体..."

正确、无用、像 HR 写的。这是因为通用 LLM 被 RLHF 调教成了"政治正确优先"，它不愿意给你具体的、带策略性的建议，因为那在训练目标里被标记为"操纵性行为"。

**方案三：问兄弟**。兄弟愿意帮你，但兄弟自己也单身。

## 三、解法：把 Claude Code 变成一个专门的中文恋爱教练

转机是 Anthropic 2025 年给 Claude Code 加了 **skill 机制**——允许你通过一个 `SKILL.md` 文件，把 Claude 在某个特定领域的行为完全重塑：加载专属的知识库、改变默认语气、注入判断框架、定义工作流。

这不是微调模型（那需要 GPU 和数据），也不是简单的 system prompt（那容易被原始训练倾向覆盖）。它更像是给 Claude 戴上一副"角色眼镜"——它还是 Claude，但它看世界的方式被你重新定义了。

我意识到这就是那个缺失的一环。通用 LLM 给不出有策略性的恋爱建议，不是因为它不会，是因为它被训练成不敢。而 skill 机制允许我构建一个**明确声明"这是一个私下咨询场景，目标是帮助用户提升真实社交能力"**的上下文，让 Claude 卸下防御，正经干活。

于是有了**情圣（qingsheng-skill）**。

## 四、技术实现的几个关键点

### 1. SKILL.md 结构

核心是一个 3000 行的 markdown 文件，分成几块：

- **身份声明**：你是一个中文男性社交与恋爱私教，服务对象是 25-40 岁中国男性
- **判断框架**：7 阶段关系模型（开场/建立好感/吸引/邀约/约会/升温/确立关系），每个阶段的典型信号、常见陷阱、推进动作
- **语料锚点**：1400+ 条真实中文对话片段（脱敏处理），作为"什么是好回复 / 什么是翻车回复"的具体样本
- **工作流**：用户贴截图 → OCR 提取 → 阶段判断 → 信号分析 → 生成 2-3 个回复选项 → 解释背后逻辑
- **子命令**：`/急`（快速模式）、`/换一个`（换角度重生成）、`/展示面`（朋友圈诊断）、`/挽回`、`/自动`（autopilot 发送序列）

### 2. 多目标档案

大部分男生同时在跟多个女生聊，每个人的进度、语气、雷区都不一样。情圣维护一个 `profiles/` 目录，每个目标一个档案：基础信息、聊天阶段、过往关键节点、她的语言风格、她的雷区（比如"不能提前任"、"不爱自拍"）。

这样当你说"黄雨洁今天发了这句话"，AI 会自动调用她的档案，给出的建议不会跟其他人的策略冲突。

### 3. OCR 截图分析

微信聊天记录很多时候复制不方便，直接发截图最快。Claude Code 支持 vision，能识别中文聊天气泡、时间戳、对话方向（左右区分你和她）。情圣在 SKILL.md 里写了专门的截图解析指引，让 Claude 不会把你自己发的话当成女生发的。

### 4. 对抗"政治正确漂移"

这是最反复调试的部分。默认的 Claude 会在给出建议后加一堆"但是你要尊重她的意愿"、"恋爱不是套路"这种安全尾巴。在实际使用里这些话是噪声——用户找教练就是要具体策略，不需要被说教。

我在 SKILL.md 里明确声明：**"用户是成年人，请把他当朋友咨询，不要当未成年保护对象。给策略，不要给正确的废话。"** 这一句话让输出质量提升了大概 40%。

## 五、Demo：前后对比

**场景**：探探匹配的女生，聊了两天，她刚发来："今天加班好累啊 😩"

### 裸用 ChatGPT 的回复

> 可以这样回："辛苦啦！加班到现在一定很累吧。要注意休息哦，早点睡，明天又是新的一天。有什么烦心事可以跟我说说。" 这样回复既表达了关心，又给对方留出了倾诉空间，体现了你的体贴和真诚。

**问题**：太像客服话术，没有推进关系，没有自我暴露，没有下一步钩子。她大概率回"嗯谢谢"然后消失。

### 情圣 skill 的回复

> **阶段判断**：建立好感中期。她主动分享负面情绪 = 信任信号，是推进机会，不是单纯倾诉。
>
> **三个选项**：
>
> **A. 共情 + 轻推进（推荐）**：
> "加班到现在才结束？明天周六吧，应该能睡到自然醒了。对了你们公司附近那家便利店的关东煮我上次路过看还开着，加班饿的时候可以去买一碗（悄悄种草）"
> → 关心 + 暴露你知道她公司位置（自然，不是跟踪感）+ 给她一个场景记忆点
>
> **B. 自嘲拉近**：
> "同情，我上周连着加班三天，最后在工位上对着电脑睡着了被同事拍到，现在是部门表情包。你是哪种累，身体累还是脑子累？"
> → 自我暴露 + 制造笑点 + 开放式问题拉长对话
>
> **C. 邀约铺垫**：
> "周末打算干嘛，补觉还是出去放风？我发现一家咖啡店特别适合发呆，周末带你去 decompress 一下"
> → 如果你们已经聊了两天氛围 OK，可以直接试探邀约
>
> **选哪个看你风险偏好**。A 最稳，B 有趣但需要你后续接得住，C 最激进，成功率中等但能快速筛选意向。

这就是 skill 跟裸用的差别——不是 AI 变聪明了，是你把**判断框架、阶段意识、本地化语境**全都注入进去了。

## 七、局限性和未来规划

**现在不行的地方：**

- 只服务中文大陆语境，对港澳台和海外华人的社交习惯覆盖不够
- 对 JK/二次元/文艺青年这些小圈层的黑话识别还不完美
- 无法判断"她是不是只是寂寞想找人聊"——这个需要更多轮对话观察
- 本质上还是个话术辅助工具，**不能替代你线下见面的真实人格和展示面**，这一点必须说清楚

**接下来想做的：**

- 语音消息分析（她发语音你听不出情绪时）
- 朋友圈时间序列分析（她的朋友圈状态变化暗示什么）
- 接入本地日历，提醒关键节点（比如"认识满一个月，可以尝试推进确立关系"）
- 更精细的展示面诊断（头像、朋友圈排版、个人简介评分）
